# 角色技能合理性审计与升级方案

更新时间：2026-06-15

## 1. 结论

当前技能系统的主要问题不是数值单点不合理，而是「角色描述很丰富，执行逻辑很通用」。

现在技能配置里写了很多有角色特色的描述，比如「瞬移到敌人身后」「全屏代码雨攻击」「创建障碍物」「投掷文件持续伤害」，但战斗引擎实际只按五类通用逻辑处理：

- `melee`：近战矩形判定，命中后扣血。
- `ranged`：生成一个横向飞行弹道，命中后扣血。
- `buff`：只有 `effect: shield` 时才生效。
- `heal`：固定回复 30 HP。
- `debuff`：降低敌方攻击倍率。

因此会出现：

- 描述和实际不符。
- 某些技能看起来释放了，但没有实际效果。
- 本地对战和在线对战同一个技能行为不一致。
- 视觉表现区分度不足，很多技能只是换了一个弹道 emoji 或弹出文字。

建议采用两阶段方案：

1. 先修复当前明确 bug：描述不符、未生效、本地/在线不一致。
2. 再升级技能模型：从 `type + effect` 扩展为明确的 `mechanic`，让每个技能有独立玩法和视觉表现。

## 2. 当前技能执行链路

### 2.1 本地对战

涉及文件：

- `D:\allproject\minigame\src\data\characters.ts`
- `D:\allproject\minigame\src\game\engine.ts`
- `D:\allproject\minigame\src\game\GameState.ts`
- `D:\allproject\minigame\src\components\BattleArena.tsx`

本地技能流程：

1. `characters.ts` 定义技能名称、描述、伤害、蓝耗、冷却、类型、效果。
2. `BattleArena.tsx` 监听键盘。
3. `tick()` 中检测按键：
   - 玩家1：`J/K/L/I`
   - 玩家2：`1/2/3/0`
4. 普攻调用 `performAttack()`。
5. 技能调用 `performSkill()`。
6. `performSkill()` 根据 `skill.type` 进入不同分支。
7. 视觉上通过粒子、弹道 emoji、技能弹字、伤害数字表现。

### 2.2 在线对战

涉及文件：

- `D:\allproject\backendMinigame\src\game\characters.ts`
- `D:\allproject\backendMinigame\src\game\engine.ts`
- `D:\allproject\backendMinigame\src\game\state.ts`
- `D:\allproject\minigame\src\components\BattleArenaOnline.tsx`
- `D:\allproject\minigame\src\net\protocol.ts`
- `D:\allproject\backendMinigame\src\protocol\types.ts`

在线技能流程：

1. 前端采集按键，发送 `battle:input`。
2. 后端 `tickGame()` 权威计算移动、普攻、技能、弹道、胜负。
3. 后端广播 `battle:snapshot` 和 `battle:events`。
4. 前端 `BattleArenaOnline.tsx` 根据快照和事件播放视觉反馈。

在线模式比本地模式多了持续帧状态：

- `shieldFrames`
- `stunFrames`
- `debuffFrames`

这比本地使用 `setTimeout()` 更稳定，也更适合联机同步。

## 3. 技能模型当前短板

### 3.1 `type` 过粗

现在 `type` 只有：

- `melee`
- `ranged`
- `buff`
- `heal`
- `debuff`

但实际设计需要表达更多机制：

- 冲刺
- 瞬移
- 多段弹道
- 追踪弹
- 全屏 AOE
- 持续伤害
- 减速
- 眩晕
- 护盾
- 治疗
- 墙体/障碍物
- 拉近/击退
- 增伤/减伤
- 霸体/无敌

仅靠 `type + effect` 很难描述这些。

### 3.2 `effect` 过少且执行不完整

前端本地支持的 `effect` 类型：

- `knockback`
- `stun`
- `shield`
- `crit`

后端在线支持：

- `knockback`
- `stun`
- `shield`
- `crit`
- `weaken`

问题：

- 本地 `Skill.effect` 类型没有 `weaken`，但张总「加薪诱惑」配置是 `debuff` 且没有 `effect`。
- 本地 `debuff` 分支不检查具体 effect，任何 `debuff` 都会把敌人伤害倍率设为 `0.6`。
- 本地 stun 对近战生效，但远程弹道没有保存 `effect`，因此远程 stun 技能命中后不会眩晕。
- 在线弹道保存了 `effect`，远程 stun 在线会生效。

### 3.3 技能描述没有被机制约束

技能描述是自由文本，和实际逻辑没有校验关系。

例如：

- 描述写「瞬移到敌人身后」，逻辑没有瞬移。
- 描述写「全屏 AOE」，逻辑只是一个普通横向弹道。
- 描述写「持续伤害」，逻辑只造成一次命中伤害。
- 描述写「创建障碍物」，本地没有障碍物实体。

这会导致玩家对技能预期和实战反馈严重不一致。

### 3.4 本地和在线技能逻辑不一致

典型差异：

- 阿杰普攻 `名片飞射`：
  - 本地：`performAttack()` 只是把 ranged 普攻当 300 范围瞬间判定。
  - 在线：服务端会生成弹道。
- 远程 stun：
  - 本地：弹道没有 `effect` 字段，命中只扣血。
  - 在线：弹道带 `effect`，命中会眩晕。
- buff / debuff 持续时间：
  - 本地：用 `setTimeout()`。
  - 在线：用帧计数。

本地使用 `setTimeout()` 还会有暂停、重开、页面切换后的状态残留风险。

## 5. 明确 Bug 清单

### P0-1：本地远程 stun 未生效

问题：

- 小明 `系统崩溃`、小红 `财务审计` 都配置了 `effect: 'stun'`。
- 本地 `Projectile` 没有 `effect` 字段。
- 本地 `performSkill()` 创建远程弹道时没有写入 `effect`。
- 本地 `tickProjectiles()` 命中后只扣血，不处理 stun。

影响：

- 技能描述/配置和实际效果不符。
- 本地与在线表现不一致。

建议：

- 给本地 `Projectile` 增加 `effect?: Skill['effect']`。
- 创建弹道时写入 `effect: skill.effect`。
- 弹道命中时处理 `stun`、`knockback`、`crit` 等效果。

### P0-2：美美「会议室预定」本地未生效

问题：

- 前端 `characters.ts` 中 `会议室预定` 是 `type: 'buff'`，但没有 `effect: 'shield'`。
- 本地 `buff` 分支只处理 `skill.effect === 'shield'`。
- 因此该技能释放后只扣蓝、进冷却、播放泛用弹字和粒子，实际没有 buff。

在线差异：

- 后端 `characters.ts` 中 `room_booking` 有 `effect: "shield"`，在线模式会生效。

建议：

- 短期给前端配置补 `effect: 'shield'`，并把描述改成「预定会议室形成临时护盾」。
- 如果坚持“障碍物”，需要新增障碍物系统。

### P0-3：阿杰「名片飞射」本地/在线不一致

问题：

- 本地 `performAttack()` 对 ranged 普攻使用 300 范围瞬间判定。
- 在线 `handleActions()` 对 ranged 普攻调用 `addProjectile()`，会生成弹道。

建议：

- 本地 `performAttack()` 如果 `attack.type === 'ranged'`，也生成弹道。
- 或统一改为普通攻击不允许 ranged，把阿杰普攻改成 melee。

更推荐前者，因为「名片飞射」是角色特色。

### P0-4：技能持续状态本地使用 `setTimeout()`

问题：

- 本地护盾、眩晕、debuff 使用 `setTimeout()` 结束。
- 这不受游戏帧控制，重开、暂停、切屏时容易残留或错乱。

建议：

- 本地也改成帧计数：
  - `shieldFrames`
  - `stunFrames`
  - `debuffFrames`
- 和在线服务端保持同一模型。

## 6. 技能丰富度升级方向

### 6.1 建议新增技能机制字段

当前：

```ts
type Skill = {
  type: 'melee' | 'ranged' | 'buff' | 'heal' | 'debuff';
  effect?: 'knockback' | 'stun' | 'shield' | 'crit';
}
```

建议扩展为：

```ts
type SkillMechanic =
  | 'strike'
  | 'projectile'
  | 'multiProjectile'
  | 'dashStrike'
  | 'teleportStrike'
  | 'aoe'
  | 'fallingAoE'
  | 'shield'
  | 'invincible'
  | 'heal'
  | 'weaken'
  | 'dot'
  | 'wall'
  | 'tornado';

type SkillEffect =
  | 'knockback'
  | 'stun'
  | 'crit'
  | 'slow'
  | 'weaken'
  | 'dot'
  | 'shield';

interface Skill {
  name: string;
  description: string;
  damage: number;
  mpCost: number;
  cooldown: number;
  type: 'melee' | 'ranged' | 'buff' | 'heal' | 'debuff';
  mechanic: SkillMechanic;
  effect?: SkillEffect;
  durationFrames?: number;
  hitCount?: number;
  projectileCount?: number;
  areaRadius?: number;
  dashDistance?: number;
  tickDamage?: number;
}
```

好处：

- 文案和逻辑能对应。
- 不同角色能真正有差异。
- 本地和在线可以共享一份技能执行器。
- 后续 UI 可以根据 `mechanic` 自动展示技能标签。

### 6.2 推荐技能机制库

#### `projectile`

单个直线弹道。

适合：

- 绩效谈话
- 报销单据

视觉：

- 拖尾
- 命中爆点
- 根据角色颜色变换弹道皮肤

#### `multiProjectile`

多个弹道连发或扇形发射。

适合：

- 代码注入
- 裁员风暴的短期版本

视觉：

- 3-5 个弹道错位发射
- 弹道之间有时间差

#### `dashStrike`

角色向前突进并攻击。

适合：

- 死机冲击
- 热情推销

视觉：

- 残影
- 地面冲刺线
- 命中时强屏震

#### `teleportStrike`

瞬移到敌人身后并攻击。

适合：

- 会议室召唤

视觉：

- 原地会议门/传送门
- 敌人背后出现会议桌图标

#### `fallingAoE`

从上方落下多个区域伤害。

适合：

- 裁员风暴
- 系统崩溃

视觉：

- 文件/代码块从天而降
- 地面提前出现预警圈

#### `aoe`

以自己或目标为中心的大范围瞬间伤害。

适合：

- 财务审计
- 组织团建

视觉：

- 全屏闪白/扫描线
- 地面扩散圆环

#### `dot`

命中后持续伤害。

适合：

- 报销单据

视觉：

- 目标头上状态图标
- 周期性小伤害数字

#### `wall`

创建障碍物。

适合：

- 会议室预定

逻辑：

- 阻挡弹道。
- 可阻挡角色移动，或仅阻挡弹道。
- 持续 `durationFrames` 后消失。

视觉：

- 半透明会议室门牌/隔断墙。

#### `tornado`

慢速大体积弹道，可多段伤害/轻微吸附。

适合：

- 签单时刻

视觉：

- 旋转合同纸。
- 命中时持续卷动。

## 7. 建议重做后的角色技能定位

### 7.1 小明：远程控制 + 爆发

- 普攻：键盘敲击，近战快速攻击。
- 技能1：代码注入，3 发代码弹连射。
- 技能2：死机冲击，向前突进并击退。
- 终极：系统崩溃，代码雨大范围落下，命中眩晕。

### 7.2 Emily：中距离消耗 + 防御

- 普攻：文件夹拍打，近战攻击。
- 技能1：绩效谈话，声波远程弹道，轻微击退。
- 技能2：团队优化，生成护盾，持续减伤。
- 终极：裁员风暴，多份文件从天而降。

- 普攻：威严拍桌，近战重击。
- 技能1：会议室召唤，瞬移到敌人背后并拍桌击退。

- 普攻：名片飞射，低伤害快速弹道。

- 技能1：会议室预定，创建会议室隔断，阻挡弹道。

## 8. 视觉体验升级方案

### 8.1 技能释放前摇

当前释放技能后只有弹字和少量粒子。建议增加前摇层：

- 角色身上出现聚光/能量环。
- 技能名以角色颜色弹出。
- 终极技能有更强的停顿和屏震。

### 8.2 技能命中反馈

建议按效果类型区分：

- 暴击：金色大数字 + 爆裂环。
- 眩晕：目标头顶旋转星星 + 蓝白电流。
- 护盾：角色周围持续护罩 + 被击中时闪烁。
- 减攻：目标身上紫色向下箭头。
- 持续伤害：目标头顶状态图标 + 小数字周期跳出。
- 治疗：绿色上升数字 + 叶片/茶杯粒子。

### 8.3 弹道视觉

当前弹道大多是 emoji，识别度有限。

建议：

- 使用 `projectile.kind` 控制渲染。
- 不同弹道拥有不同形状：
  - 代码弹：绿色字符块。
  - 声波：粉色扩散弧。
  - 文件：白色纸张飞行。
  - 名片：橙色卡片旋转。
  - 合同龙卷风：旋转纸片团。

### 8.4 终极技能表现

终极技能建议统一有三段：

1. 释放提示：屏幕边缘染色，技能名大字。
2. 主效果：大范围粒子/弹道/区域。
3. 结果反馈：命中数字、控制状态、屏震。

### 8.5 状态 UI

建议在角色血条下方显示小状态：

- 护盾：`SHIELD 2.3s`
- 眩晕：`STUN 1.2s`
- 减攻：`WEAKEN 3.5s`
- 持续伤害：`DOT 2.0s`

在线模式可以从服务端快照增加剩余帧，前端转秒显示。

## 9. 本地与在线统一方案

### 9.1 抽共享战斗核心

当前本地和后端各有一套 `engine.ts` 和 `characters.ts`。短期可以同步修，长期建议抽共享包：

```text
shared/
  game/
    characters.ts
    engine.ts
    state.ts
    skillExecutor.ts
    constants.ts
```

前端本地模式和后端在线模式都引用同一套规则，减少双份逻辑漂移。

### 9.2 避免本地使用 `setTimeout`

所有持续效果都应由帧推进：

```ts
fighter.statuses = {
  shield: 180,
  stun: 90,
  weaken: 240,
  invincible: 90,
};
```

每 tick 递减。这样本地和在线完全一致，也便于快照同步。

### 9.3 扩展事件协议

当前在线事件只有：

- `hit`
- `skill`
- `gameOver`

建议增加：

```ts
type GameEvent =
  | { type: 'skillCast'; owner; skillId; mechanic; x; y }
  | { type: 'hit'; target; damage; crit; effect? }
  | { type: 'statusApplied'; target; status; durationFrames }
  | { type: 'heal'; target; value }
  | { type: 'projectileSpawn'; owner; projectileKind; x; y }
  | { type: 'aoe'; owner; areaKind; x; y; radius }
  | { type: 'gameOver'; winner };
```

前端就能根据事件精准播放视觉效果，而不是只靠技能名猜。

## 11. 推荐实施顺序

### 第一步：低风险修复

目标：不大改架构，先解决“技能没生效/本地在线不一致”。

改动：

- 前端本地 `Projectile` 增加 `effect`。
- 本地弹道命中处理 `stun`。
- 美美 `会议室预定` 补 `effect: 'shield'`。
- 阿杰远程普攻改为本地弹道。
- 更新技能描述，避免全屏/持续/瞬移等尚未实现的词。

### 第二步：帧状态重构

目标：让本地和在线持续效果一致。

改动：

- 本地 `FighterState` 增加 `shieldFrames`、`stunFrames`、`debuffFrames`。
- 删除本地技能里的 `setTimeout()`。
- 在 tick 中递减状态帧。
- UI 根据状态显示持续反馈。

### 第三步：机制升级

目标：让技能真正变丰富。

先做四个机制：

- `dashStrike`
- `multiProjectile`
- `aoe`
- `dot`

对应角色：

- 小明：代码注入多弹道，死机冲击突进。
- 小红：报销单据持续伤害，财务审计 AOE。
- 阿杰：热情推销三段突进，签单时刻龙卷风。
- Emily：裁员风暴落下文件。

### 第四步：在线协议升级

目标：让视觉不再靠猜。

改动：

- 后端事件增加 mechanic/effect/status 信息。
- 前端根据事件播放精确特效。
- 快照增加状态剩余帧。

## 12. 验收清单

### 功能验收

- 小明 `系统崩溃` 本地命中后能眩晕。
- 小红 `财务审计` 本地命中后能眩晕。
- 美美 `会议室预定` 本地释放后有实际效果。
- 阿杰 `名片飞射` 本地和在线都是弹道。
- 张总 `加薪诱惑` 命中/释放后有明确减攻状态提示。
- 护盾、眩晕、减攻不会在重开后残留。

### 描述验收

- 每个技能描述都能在战斗中观察到对应效果。
- 未实现的能力不出现在描述中，比如尚未实现前不写“全屏”“持续”“瞬移”“障碍物”。
- 技能面板显示伤害、蓝耗、冷却、机制、效果、持续时间。

### 视觉验收

- 普通技能和终极技能视觉强度有明显差异。
- 暴击、眩晕、护盾、治疗、减攻有不同视觉语言。
- 弹道类型可凭视觉区分，而不只是颜色不同。
- 在线模式下命中数字、技能名、状态反馈和服务端事件一致。

### 一致性验收

- 本地和在线同角色同技能效果一致。
- 同一套技能配置能驱动本地和服务端。
- 修改技能数值不需要同时改两份配置。
